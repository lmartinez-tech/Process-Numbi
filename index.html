Métrica de éxito

NPS 9+ significa claridad y acompañamiento

La calificación final no depende solo de que la contabilidad quede técnicamente correcta. Depende de cómo vivió el cliente el proceso.

Qué mide el cliente

Su experiencia

  • Si se sintió acompañado.
  • Si entendió qué ocurría en cada momento.
  • Si tuvo claridad sobre tiempos y expectativas.
  • Si percibió orden y buen servicio.
Qué no mide

No es una auditoría técnica

  • Qué tan rápido fue el setup.
  • Qué tan perfecta quedó la contabilidad.
  • Cuántas funciones del portal utilizó.
Por qué importa

La claridad cambia la nota

Un setup técnicamente sólido puede obtener una mala calificación si el cliente estuvo desinformado. La comunicación visible convierte el trabajo técnico en confianza.

Objetivo de la guía

¿Qué vamos a hacer?

Vamos a ordenar el setup contable para que el equipo sepa qué revisar, qué preguntar, cuándo escalar y cuándo pasar a Operación sin dejar cabos sueltos.

Lectura ejecutiva El proceso no debe sentirse como una auditoría pesada para el cliente. Debe sentirse como acompañamiento: diagnóstico claro, ruta explicada, avances visibles, riesgos conversados a tiempo y cierre formal antes de iniciar la operación recurrente.
Mapa de participación

¿Quién participa y en qué momento?

La idea es que cada persona entienda cuándo entra, qué debe preparar y qué decisión ayuda a tomar. Así el proceso se siente acompañado, no como una lista interminable.

Fase 1 · Onboarding

Fase 1 — Pre-onboarding y Onboarding

El gerente no llega a descubrir al cliente desde cero. Llega con una hipótesis inicial y usa la reunión para validarla.

A. Antes de la reunión

Pre-onboarding: preparación del gerente

Inicia cuando Mayerly agenda la reunión, crea el grupo de WhatsApp con el cliente y agrega al equipo correspondiente.

  • Revisar Acta de Kick Off en la sección de Onboarding del Dashboard de Migración.
  • Revisar notas o transcripción de la reunión comercial.
  • Revisar Propuesta Comercial del cliente.
  • Entender actividad, tamaño, software actual, expectativas, riesgos y compromisos.
  • Construir un diagnóstico inicial antes de la reunión.
B. Durante la reunión

Onboarding: validación de la hipótesis

La reunión sirve para validar o corregir la hipótesis inicial, no para empezar desde cero.

  • Abrir con una lectura preparada del cliente.
  • Validar tipo de operación, tipo de cliente y estado contable.
  • Identificar riesgos contables, tributarios y operativos.
  • Definir focos obligatorios para el contador designado.
  • Dejar clara la ruta de Set Up que aplica.
Idea clave El onboarding no busca descubrir desde cero qué hace el cliente. Eso ya debió investigarse. El onboarding valida cómo opera, cómo factura, cómo cobra, cómo paga, qué tan confiable es su pasado contable y qué debe auditar obligatoriamente el contador designado.
Rutas posibles

Qué está clasificando el gerente

La ficha de diagnóstico inicial nace en el pre-onboarding. Estas son las rutas que el gerente puede anticipar antes de la reunión.

Confirmación

Cuando el pasado es confiable o no existe

  • Cliente nuevo: sin pasado contable; se configura para empezar operación.
  • Software compatible: trae pasado en Siigo o Alegra; se validan saldos antes de operar.
  • Migración / parametrización: requiere configurar o migrar información.
Auditoría

Cuando hay dudas sobre la calidad contable

  • Software compatible: se audita la información histórica antes de decidir.
  • Migración / parametrización: se audita y probablemente se configura o migra.
Reconstrucción

Cuando el pasado no es confiable

  • Software compatible: se reconstruye sin necesidad de migrar software.
  • Migración / parametrización: se reconstruye y además se migra o parametriza.
Categorías base

Cómo leer al cliente

  • Operación: servicios, comercial, industrial o mixta.
  • Tipo de cliente: empresa nueva, software compatible o migración / parametrización.
  • Estado contable: confirmación, auditoría o reconstrucción.
Ficha de diagnóstico inicial

El resultado práctico del pre-onboarding

Antes de la reunión, el Gerente Contador empieza a completar la ficha con cliente, tipo de operación, tipo de cliente, estado contable, software actual, software destino, hipótesis inicial, riesgo principal, cuentas críticas, ruta probable, preguntas clave y observaciones.

Explicación de setup

El Set Up se activa según la ruta

El Set Up es la ejecución técnica y contable que nace de la clasificación del onboarding. No todos los clientes pasan por todas las fases: algunas aplican siempre y otras dependen del tipo de cliente, el estado contable, la migración, la parametrización y los riesgos detectados.

Quién lidera

Gerente Contador

Lidera el proceso según el tipo de caso, decide la ruta y asegura que el contador designado reciba focos claros.

Quién no participa por defecto

Ventas y Country Manager

Ventas no participa en el Set Up. El Country Manager solo se involucra cuando existe una alerta.

Qué activa la ruta

Tipo de cliente + estado contable

El Set Up no es una lista única igual para todos; es una ruta según la situación real del cliente.

Fase 1

Onboarding

Clasificar al cliente, definir la ruta y dejar acuerdos. Es la puerta de entrada para activar el Set Up correcto.

Fase 2

Recepción y validación

Primero se recibe la documentación y se revisa si está completa. Si falta algo, se notifica y se deja la base lista para auditar o confirmar.

Fase 3

Auditoría

Se revisa en profundidad cuando hay dudas sobre el estado real de la contabilidad. El resultado puede ser continuidad, ajustes o reconstrucción.

Fase 4

Parametrización y migración

Se configura Siigo o Alegra y se migra la información cuando el cliente lo necesita. El objetivo es dejar el sistema listo para operar.

Fase 5

Reconstrucción contable

Se reconstruyen periodos anteriores cuando el pasado contable no es confiable. Aquí la prioridad es recuperar orden, evidencia y trazabilidad.

Fase 6

Salida a operación

Se capacita al cliente, se cierra el setup y se cambia formalmente a operación recurrente cuando el checklist mínimo está completo.

Validaciones por foco
Matriz de focos obligatorios

Qué debe revisar el contador

  • Si hay inventarios: inventario físico, costo de ventas, ajustes y saldos iniciales.
  • Si hay producción: materia prima, producto en proceso, producto terminado, MOD y CIF.
  • Si hay proyectos: centros de costo, ingresos por proyecto y costos asociados.
  • Si hay anticipos: pasivos por anticipos e ingresos reconocidos.
  • Si hay ventas a crédito: cartera, provisión, recaudos y retenciones.
  • Si hay importaciones: proveedores del exterior, TRM, nacionalización e impuestos.
Alertas de foco

Cuándo escalar la revisión

  • Si hay nómina operativa: separar costo y gasto.
  • Si hay cuentas personales de socios: revisar bancos, socios, préstamos y gastos sin soporte.
  • Si hay impuestos pendientes: declaraciones, pagos, sanciones y conciliación contable.
  • Si hay software anterior: saldos iniciales, auxiliares y exportación de información.
  • Si la contabilidad no es confiable: activar auditoría de saldos o reconstrucción.
